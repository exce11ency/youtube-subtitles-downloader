index.html – это основа любой веб-страницы. Он содержит:
HTML (HyperText Markup Language): Определяет структуру страницы (заголовки, кнопки, поля ввода, текст).
CSS (Cascading Style Sheets): Отвечает за внешний вид (цвета, шрифты, расположение элементов). В нашем случае мы используем "Tailwind CSS" через CDN, что упрощает стилизацию.
JavaScript (JS): Добавляет интерактивность. В этом файле JavaScript отвечает за:
Получение URL YouTube видео, которое вводит пользователь.
Извлечение ID видео из этого URL.
Показ загрузочного индикатора и скрытие ошибок.
(ВАЖНО) Он содержит закомментированный код для отправки запроса на ваш будущий сервер (бэкенд) и симуляцию ответа от него. Именно здесь мы будем вносить изменения для подключения к реальному бэкенду.

requirements.txt – это текстовый файл, в котором перечислены все внешние библиотеки (зависимости), необходимые для работы вашего Python-проекта. Когда вы или кто-то другой будет запускать ваш код, этот файл позволит быстро установить все нужные компоненты.

app.py – это основной файл вашего серверного приложения на Python. Он будет:

Принимать запросы от вашего index.html (когда пользователь нажимает "Fetch Subtitles").
Извлекать ID видео из запроса.
Использовать youtube-transcript-api для получения доступных субтитров для этого видео.
Форматировать эти субтитры (при необходимости, например, в TXT или SRT).
Отправлять полученные субтитры обратно в ваш index.html.
