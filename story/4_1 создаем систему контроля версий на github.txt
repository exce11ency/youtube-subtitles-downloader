ля публикации веб-приложения на хостинге, таком как Render, вам нужно, чтобы ваш код был в системе контроля версий, обычно это Git, а затем размещен на платформе, такой как GitHub.

Что такое Git и GitHub?

Git – это система контроля версий. Она позволяет вам отслеживать изменения в вашем коде, возвращаться к предыдущим версиям, работать в команде и т.д.
GitHub – это веб-сервис для хостинга Git-репозиториев. Это как облачное хранилище для вашего кода, где вы можете делиться им, сотрудничать и откуда Render.com сможет забирать ваш проект.
Что вам нужно сделать:

Создать аккаунт на GitHub (если у вас его нет):

Перейдите на github.com и зарегистрируйтесь. Это бесплатно для публичных репозиториев.
Установить Git на ваш компьютер (если он еще не установлен):

Для Windows: Скачайте и установите Git с git-scm.com/download/win. Следуйте инструкциям установщика, выбирая параметры по умолчанию.
Для macOS: Git часто уже предустановлен. Откройте терминал и введите git --version. Если он не установлен, macOS предложит установить его, или вы можете использовать Homebrew: brew install git.
Для Linux: Используйте менеджер пакетов вашей системы, например: sudo apt install git (для Debian/Ubuntu) или sudo yum install git (для Fedora/CentOS).
Инициализировать Git-репозиторий в папке вашего проекта:

Откройте терминал/командную строку.
Перейдите в папку вашего проекта youtube_subs_downloader (где лежат index.html, app.py, requirements.txt):
Bash

cd /путь/к/вашей/папке/youtube_subs_downloader
Инициализируйте новый Git-репозиторий:
Bash

git init
Вы увидите сообщение типа Initialized empty Git repository in /path/to/your/project/.git/.
Создать файл .gitignore:
Этот файл говорит Git, какие файлы и папки не нужно отслеживать и загружать на GitHub. Это важно, потому что некоторые файлы (например, папки с виртуальными окружениями Python) не нужны на сервере или могут содержать конфиденциальную информацию.
Создайте новый файл с именем .gitignore (обязательно с точкой в начале) в корневой папке вашего проекта (youtube_subs_downloader/). Скопируйте в него следующее содержимое:

Git Ignore File
Jun 22, 2:02 AM
* `env/`, `venv/`: Это стандартные имена для папок виртуальных окружений Python, которые не должны попадать в репозиторий.
* `__pycache__/`: Папка с временными файлами Python.
Добавить файлы в Git и сделать первый коммит:
Теперь, когда Git инициализирован и .gitignore создан, добавьте все ваши файлы в репозиторий:

Bash

git add .
(Обратите внимание на точку . – она означает "добавить все текущие файлы и папки").

Затем сделайте первый "коммит" (снимок состояния вашего проекта):

Bash

git commit -m "Initial project setup: frontend, backend, requirements"
Сообщение -m "..." – это краткое описание ваших изменений.

Создать новый репозиторий на GitHub:

Зайдите на github.com.
На главной странице GitHub (или в правом верхнем углу) нажмите кнопку "New" (или "+") для создания нового репозитория.
Дайте ему имя, например, youtube-subtitles-downloader (без пробелов, используйте дефисы).
Сделайте его Public (публичным), чтобы Render мог получить к нему доступ.
Не ставьте галочки "Add a README file", "Add .gitignore", "Choose a license", так как мы уже сделали это локально.
Нажмите "Create repository".
Связать локальный репозиторий с удаленным на GitHub и отправить код:
После создания репозитория на GitHub, вы увидите инструкции. Вам нужны две последние команды:

Bash

git remote add origin https://github.com/ВАШЕ_ИМЯ_ПОЛЬЗОВАТЕЛЯ/youtube-subtitles-downloader.git
git branch -M main
git push -u origin main
Замените ВАШЕ_ИМЯ_ПОЛЬЗОВАТЕЛЯ на ваш никнейм на GitHub.
Вам может потребоваться ввести свои учетные данные GitHub (имя пользователя и пароль или токен доступа), когда вы выполните git push.
После выполнения этих шагов ваш проект будет успешно загружен на GitHub!
